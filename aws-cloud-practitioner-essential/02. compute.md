# dedicated instance vs dedicated host

- dedicated instance: hardware dedicated to your AWS account, but your *other* instances may share the same physical host. no visibility/control over the host. host can change on stop/start.
- dedicated host: you own the entire physical server. visibility and control over instance placement. needed for BYOL (per-socket/per-core licensing like Windows Server, Oracle, SQL Server), and for compliance requirements that mandate a specific physical server. billed per host.

| feature | dedicated instance | dedicated host |
|---|---|---|
| hardware isolated from other AWS accounts | yes | yes |
| sole tenant of the entire physical server | no (your other instances may share it) | yes |
| visibility into the physical host | no | yes |
| control over instance placement | no | yes |
| BYOL with per-socket/core licensing | no | yes |
| billing | per instance | per host |

exam tip
- per-socket / per-core / BYOL / compliance / visibility into host => dedicated host
- hardware dedicated to a single account, no host visibility needed => dedicated instance
- share with other AWS customers => shared tenancy (default)
