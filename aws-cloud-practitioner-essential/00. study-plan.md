### Wed May 6 — Global infrastructure + Networking (your IT-shaky area)

Plain-English IT primer first (15 min): A **VPC** is your own private LAN inside AWS. A **subnet** is a slice of that LAN that lives in one data center (AZ). **Public subnet** = has a route to the internet via an **Internet Gateway**. **Private subnet** = no direct internet; needs a **NAT Gateway** to *call out* (e.g. download patches) but nothing can call *in*. **Route 53** = DNS (turns names into IPs). **CloudFront** = caches your content at hundreds of edge locations close to users so it loads fast.

- [ ] `04. goingGlobal.md` — Regions vs AZs vs Edge Locations vs Local Zones vs Wavelength vs Outposts. Know "min 3 AZs per Region."
- [ ] `05. networking.md` + `05. Networking.md` (read both, dedupe in your head).
- [ ] Cheat-sheet **Networking & Content Delivery** section — quiz yourself on every line.
- [ ] Memorize the 4 ways to connect on-prem to AWS:
  1. **Site-to-Site VPN** — encrypted over public internet, fast to set up
  2. **Direct Connect** — dedicated private fiber, low latency, takes weeks
  3. **Client VPN** — individual remote users
  4. **Transit Gateway** — hub for many VPCs/on-prem
- [ ] Distractor pair: **CloudFront vs Global Accelerator** (caches content vs routes to nearest region, no caching).
- [ ] Maarek sections on **VPC** and **Global Infrastructure**.

### Fri May 8 — Storage (your other IT-shaky area)

Plain-English IT primer (15 min): **Block storage** = a raw hard disk attached to one server (EBS). **File storage** = a shared network drive multiple servers can mount (EFS, FSx). **Object storage** = upload/download whole files via API; effectively infinite, can't be mounted as a drive (S3).

- [ ] `06. Storage.md` cover to cover.
- [ ] **S3 storage classes** — must know retrieval speed + relative cost ordering: Standard → Standard-IA / One Zone-IA / Intelligent-Tiering → Glacier Instant → Glacier Flexible → Glacier Deep Archive. Look at `S3LifecycleFlow.png`.
- [ ] **EBS vs EFS vs FSx** distractor — be able to pick instantly:
  - One EC2 needs a disk → **EBS**
  - Many Linux EC2 share files → **EFS**
  - Windows file share / Lustre HPC / NetApp → **FSx**
- [ ] **Snow Family** sizes: Snowcone (~8 TB) → Snowball Edge (~80 TB) → Snowmobile (truck, exabytes).
- [ ] **Storage Gateway** types: File / Volume / Tape.
- [ ] Maarek section on **S3 + Other Storage**.
