# AWS Cloud Practitioner (CLF-C02) — Service Cheat Sheet

> One-line recognition guide. If you know the keyword → service mapping for every line below, you pass.

## Compute

- **EC2** — rentable virtual servers (VMs).
- **EC2 Auto Scaling** — automatically add/remove EC2 based on demand.
- **ELB** (ALB / NLB / GLB / CLB) — distribute traffic across targets.
  - ALB = HTTP/HTTPS, layer 7
  - NLB = TCP/UDP, layer 4, ultra-fast
  - GLB = third-party network appliances
  - CLB = legacy
- **Lambda** — run code in response to events; no servers; pay per ms.
- **Fargate** — run containers without managing servers (works with ECS/EKS).
- **ECS** — AWS's own container orchestrator.
- **EKS** — managed Kubernetes.
- **ECR** — private Docker image registry.
- **Lightsail** — simple VPS for small apps/blogs at flat price.
- **Batch** — run batch computing jobs at any scale.
- **Elastic Beanstalk** — upload code, AWS provisions the stack (PaaS).
- **Outposts** — AWS hardware racks in your own data center.
- **Wavelength** — AWS compute embedded in 5G telco networks.
- **Local Zones** — AWS infra closer to specific metros for low latency.

## Storage

- **S3** — object storage; 11 9s durability; unlimited.
- **S3 storage classes** — Standard / Standard-IA / One Zone-IA / Intelligent-Tiering / Glacier Instant / Glacier Flexible / Glacier Deep Archive. Cheaper = slower retrieval.
- **EBS** — block storage attached to a single EC2 (like a disk).
- **EFS** — managed NFS file system, multi-EC2 access, Linux only.
- **FSx** — managed file systems for Windows / Lustre / NetApp ONTAP / OpenZFS.
- **Storage Gateway** — hybrid: on-prem appliance backed by S3 (File / Volume / Tape).
- **AWS Backup** — central backup across many AWS services.
- **Snow Family** — physical data transfer.
  - **Snowcone** — handheld, ~8 TB
  - **Snowball Edge** — ~80 TB + compute
  - **Snowmobile** — truck, exabytes (being retired, still on exam)

## Database

- **RDS** — managed relational DB (MySQL / Postgres / MariaDB / Oracle / SQL Server / Aurora).
- **Aurora** — AWS's own MySQL/Postgres-compatible DB; 5x faster, cloud-native.
- **DynamoDB** — managed serverless NoSQL key-value DB; single-digit ms.
- **DocumentDB** — managed MongoDB-compatible.
- **ElastiCache** — managed Redis or Memcached (in-memory cache).
- **Neptune** — managed graph DB.
- **Keyspaces** — managed Cassandra.
- **QLDB** — ledger DB (immutable, cryptographically verifiable history).
- **Timestream** — time-series DB.
- **Redshift** — managed data warehouse for analytics over petabytes; SQL on big historical data. (OLAP, not OLTP — vs RDS.)
- **DMS** — Database Migration Service; migrate DBs to AWS, including across engines.

## Networking & Content Delivery

- **VPC** — your own private virtual network in AWS.
- **Subnet** — slice of a VPC in one AZ; public or private.
- **Internet Gateway / NAT Gateway** — outbound internet for public / private subnets.
- **Route 53** — managed DNS + domain registration + health checks.
- **CloudFront** — global CDN; caches content at edge locations.
- **Global Accelerator** — static IPs that route to nearest region (not caching).
- **API Gateway** — managed REST/HTTP/WebSocket API front door.
- **Direct Connect** — private dedicated fiber from your DC to AWS (no internet).
- **Site-to-Site VPN** — encrypted tunnel over public internet to your DC.
- **Client VPN** — remote-user VPN into AWS.
- **Transit Gateway** — hub-and-spoke router connecting many VPCs and on-prem.
- **PrivateLink** — privately expose services across VPCs without going over internet.
- **App Mesh** — service mesh for microservices.
- **Cloud Map** — service discovery for cloud resources.

## Security, Identity & Compliance

- **IAM** — users, groups, roles, policies. Free, global.
- **IAM Identity Center** (formerly SSO) — single sign-on across AWS accounts + apps.
- **Cognito** — user sign-up / sign-in for **your** apps (mobile/web).
- **Organizations** — multi-account management + consolidated billing + SCPs.
- **Control Tower** — opinionated multi-account landing zone built on Organizations.
- **KMS** — managed encryption keys (shared HSM).
- **CloudHSM** — dedicated single-tenant HSM (FIPS 140-2 Level 3).
- **Secrets Manager** — store + auto-rotate secrets (DB creds, API keys).
- **Systems Manager Parameter Store** — store config + secrets (cheaper, no rotation).
- **Certificate Manager (ACM)** — free public TLS certs for AWS services.
- **WAF** — layer-7 firewall rules (block SQLi, XSS, bad IPs).
- **Shield** — DDoS protection. Standard = free auto, Advanced = paid + 24/7 DRT.
- **GuardDuty** — threat detection from VPC flow logs, DNS, CloudTrail.
- **Inspector** — vulnerability scanning of EC2, ECR, Lambda.
- **Macie** — discovers PII / sensitive data in S3.
- **Detective** — investigates security findings (root cause).
- **Security Hub** — central dashboard of security findings across services.
- **Audit Manager** — automate compliance audits.
- **Artifact** — download AWS compliance reports (SOC, ISO, PCI).
- **Firewall Manager** — centrally manage WAF/Shield/SG rules across accounts.
- **Network Firewall** — managed network firewall for VPCs.

## Management & Governance

- **CloudWatch** — metrics, logs, alarms, dashboards.
- **CloudTrail** — records all AWS API calls (audit log). Default 90 days in Event History.
- **Config** — tracks resource configuration changes over time + compliance rules.
- **Trusted Advisor** — automated checks across 5 categories (cost, perf, security, fault tolerance, service limits). Full set requires Business/Enterprise support.
- **Well-Architected Tool** — manually review your workloads against the 6 pillars.
- **Systems Manager (SSM)** — patching, run commands, parameter store, session manager.
- **OpsWorks** — managed Chef/Puppet (legacy, still on exam).
- **CloudFormation** — Infrastructure as Code via JSON/YAML templates.
- **CDK** — IaC using real programming languages → compiles to CloudFormation.
- **Service Catalog** — let users self-serve approved IaC products.
- **License Manager** — track BYOL software licenses.
- **Compute Optimizer** — recommends right-sized EC2/EBS/Lambda based on usage.
- **Health Dashboard** — personalized + public AWS service health.
- **Launch Wizard** — guided setup for SAP, SQL Server, etc.
- **Resource Groups & Tag Editor** — group/tag resources at scale.

## Migration & Transfer

- **Migration Hub** — single dashboard tracking all migrations.
- **Application Discovery Service** — discover on-prem servers/apps before migrating.
- **Application Migration Service (MGN)** — lift-and-shift servers to AWS (replaces SMS).
- **DMS** — see Database section.
- **DataSync** — fast online file transfer to S3 / EFS / FSx.
- **Transfer Family** — managed SFTP/FTPS/FTP/AS2 fronting S3 or EFS.
- **Snow Family** — see Storage section.

## Analytics

- **Athena** — serverless SQL queries directly on S3 data; pay per query.
- **Glue** — managed ETL + data catalog (serverless).
- **EMR** — managed Hadoop / Spark / Hive / Presto clusters.
- **Kinesis Data Streams** — real-time streaming data ingest.
- **Kinesis Data Firehose** — load streaming data into S3/Redshift/OpenSearch (no code).
- **MSK** — managed Kafka.
- **OpenSearch Service** — managed Elasticsearch (search + log analytics).
- **QuickSight** — BI dashboards with ML insights.
- **Lake Formation** — quickly build a secure data lake on S3.
- **Data Exchange** — subscribe to third-party datasets.
- **Redshift** — see Database.

## Application Integration

- **SQS** — fully managed message **queue** (point-to-point, pull, durable).
- **SNS** — fully managed **pub/sub** (one-to-many push notifications, topics).
- **EventBridge** — serverless event bus that routes events between AWS services + SaaS by rules.
- **Step Functions** — serverless workflow orchestration (state machines).
- **MQ** — managed ActiveMQ/RabbitMQ (use for traditional message broker / lift-and-shift).
- **AppFlow** — no-code SaaS-to-AWS data sync (Salesforce, ServiceNow, etc.).
- **SWF** — older workflow service (legacy; Step Functions usually correct).

## Developer Tools

- **CodeCommit** — Git repos hosted in AWS (deprecated for new customers, still on exam).
- **CodeBuild** — managed build/CI service.
- **CodeDeploy** — automated deployments to EC2 / Lambda / ECS / on-prem.
- **CodePipeline** — orchestrates the full CI/CD pipeline.
- **CodeArtifact** — managed package repo (npm, PyPI, Maven, NuGet).
- **CodeStar** — wraps the above into project templates (legacy).
- **Cloud9** — browser-based IDE.
- **X-Ray** — distributed tracing for microservices.
- **CodeGuru** — ML-powered code review + profiler.

## Machine Learning / AI (recognition only)

- **SageMaker** — build/train/deploy ML models end-to-end.
- **Rekognition** — image + video analysis (faces, objects, moderation).
- **Comprehend** — NLP (sentiment, entities, key phrases).
- **Polly** — text → speech.
- **Transcribe** — speech → text.
- **Translate** — language translation.
- **Lex** — chatbots (the Alexa engine).
- **Textract** — extract text/forms/tables from scanned docs.
- **Forecast** — time-series forecasting.
- **Personalize** — real-time recommendations.
- **Kendra** — enterprise search powered by ML.
- **Bedrock** — access to foundation models / GenAI via API.
- **Q** — AWS's GenAI assistant for business + builders.

## End User Computing

- **WorkSpaces** — managed virtual desktops (DaaS).
- **AppStream 2.0** — stream individual desktop apps to a browser.
- **WorkDocs** — managed file sharing (being discontinued, still on exam).

## Business Apps / Customer Engagement

- **Connect** — managed cloud call center.
- **Pinpoint** — multi-channel marketing comms (email/SMS/push).
- **SES** — bulk transactional email.
- **Chime** — meetings + voice/video communication.

## IoT (light coverage)

- **IoT Core** — connect & manage IoT devices.
- **Greengrass** — run AWS Lambda + ML on edge IoT devices.

## Cost Management

- **Billing Dashboard** — current month spend.
- **Cost Explorer** — visualize and forecast costs over time.
- **Budgets** — alerts when costs/usage exceed thresholds.
- **Cost & Usage Report (CUR)** — most detailed billing data → S3.
- **Pricing Calculator** — estimate cost of a planned architecture.
- **Savings Plans** — commit to $/hr for 1 or 3 years; flexible across instance types/regions.
  - Compute SP > EC2 SP in flexibility
- **Reserved Instances** — commit to specific instance type/region for 1 or 3 yrs.
- **Spot Instances** — spare capacity, up to 90% off, can be reclaimed in 2 min.
- **Savings Plans vs RI** — SP is newer, simpler, more flexible; RI is older, more granular.

## Frameworks & Concepts

### Well-Architected — 6 Pillars

1. Operational Excellence
2. Security
3. Reliability
4. Performance Efficiency
5. Cost Optimization
6. Sustainability

### Cloud Adoption Framework (CAF) — 6 Perspectives

1. Business
2. People
3. Governance
4. Platform
5. Security
6. Operations

### Shared Responsibility Model

- **AWS** = security **OF** the cloud (hardware, hypervisor, facilities, managed service internals).
- **Customer** = security **IN** the cloud (data, IAM, OS patching on EC2, network config, encryption choices).

### 6 Advantages of Cloud (memorize AWS phrasing)

1. Trade capital expense for variable expense
2. Benefit from massive economies of scale
3. Stop guessing capacity
4. Increase speed and agility
5. Stop spending money running and maintaining data centers
6. Go global in minutes

### Cloud Service Models

- **IaaS** — EC2, VPC
- **PaaS** — Elastic Beanstalk, RDS, Lambda
- **SaaS** — WorkMail, Connect, Chime

### Deployment Models

- Cloud (public)
- Hybrid
- On-premises (private)

## Common "X vs Y" Distractor Pairs

- **Macie vs GuardDuty vs Inspector** — PII in S3 / threat detection / vulnerability scanning
- **KMS vs CloudHSM** — managed shared HSM / dedicated single-tenant HSM
- **SQS vs SNS vs EventBridge** — queue / pub-sub / event router with rules
- **Direct Connect vs Site-to-Site VPN** — dedicated private line / encrypted over internet
- **CloudWatch vs CloudTrail vs Config** — metrics & logs / API calls / resource state changes
- **Trusted Advisor vs Well-Architected Tool** — automated checks / manual workload review
- **Reserved Instance vs Savings Plan vs Spot** — commit to instance / commit to $/hr / spare capacity
- **Organizations SCP vs IAM policy** — account-level guardrail / identity permission
- **Aurora vs RDS** — AWS-native cloud-optimized / standard managed DB engines
- **Redshift vs RDS** — analytics warehouse (OLAP) / transactional DB (OLTP)
- **CloudFront vs Global Accelerator** — caches content / routes to nearest region (no caching)
- **WAF vs Shield** — layer-7 app firewall rules / DDoS protection
- **Secrets Manager vs Parameter Store** — auto-rotation & paid / cheap & no rotation
- **EFS vs FSx vs EBS** — multi-EC2 NFS / Windows-or-Lustre file / single-EC2 disk
- **MGN vs DMS vs DataSync vs Snow** — server lift-and-shift / database migration / online file transfer / physical transfer
- **IAM Identity Center vs Cognito** — sign-in to AWS & enterprise apps / sign-in for your customers
- **Step Functions vs SWF** — modern serverless workflows / legacy workflows
